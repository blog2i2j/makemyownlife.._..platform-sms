# 新增短信渠道适配器指南

本文档详细说明如何在 Platform SMS 系统中添加一个新的短信渠道适配器。

---

## 添加步骤

### 1. 创建新的适配器模块

在 `platform-sms-adapter` 目录下创建新的渠道模块：

```bash
platform-sms-adapter/
└── adapter-company/          # 新的渠道适配器模块
    ├── pom.xml
    └── src/
        ├── main/
        │   ├── java/
        │   └── resources/
        └── test/
            └── java/
```

---

### 2. 实现适配器接口

在 Java 源码中添加适配器类，实现 `cn.javayong.platform.sms.adapter.OuterAdapter` 接口：

```java
package cn.javayong.platform.sms.adapter.company;

import cn.javayong.platform.sms.adapter.command.resp.SmsRespCommand;

import java.util.Map;

/**
 * Company 渠道适配器
 */
@SPI("company")
public class CompanyOuterAdapter implements OuterAdapter {

    /**
     * 适配器初始化接口
     */
    @Override
    public void init(SmsChannelConfig smsChannelConfig) throws Exception {
        // 实现初始化逻辑
        // 从 smsChannelConfig 中获取配置信息
        // 例如：appId、appSecret、endpoint 等
    }

    /**
     * 按照模版编号发送短信
     */
    @Override
    public SmsRespCommand<String> sendSmsByTemplateId(SendSmsReqCommand smsSendRequest) {
        // 实现短信发送逻辑
        // 调用 Company 渠道的 API 发送短信
        SmsRespCommand<String> response = new SmsRespCommand<>();
        // 设置响应结果
        return response;
    }

    /**
     * 添加短信模板
     */
    @Override
    public SmsRespCommand<Map<String, String>> addSmsTemplate(AddSmsTemplateReqCommand addSmsTemplateReqCommand) {
        // 实现添加模板逻辑
        // 调用 Company 渠道的 API 添加模板
        SmsRespCommand<Map<String, String>> response = new SmsRespCommand<>();
        // 设置响应结果
        return response;
    }

    /**
     * 查询短信模板状态
     */
    @Override
    public SmsRespCommand<Integer> querySmsTemplateStatus(QuerySmsTemplateReqCommand querySmsTemplateReqCommand) {
        // 实现查询模板状态逻辑
        // 调用 Company 渠道的 API 查询模板状态
        SmsRespCommand<Integer> response = new SmsRespCommand<>();
        // 设置响应结果
        return response;
    }

    /**
     * 外部适配器销毁接口
     */
    @Override
    public void destroy() {
        // 实现销毁逻辑
        // 释放资源，关闭连接等
    }
```

### 3. 配置 SPI

在 `META-INF/sms/` 目录下创建 SPI 配置文件：

**文件路径：**
```
src/main/resources/META-INF/sms/cn.javayong.platform.sms.adapter.OuterAdapter
```

**文件内容：**
```properties
company=cn.javayong.platform.sms.adapter.company.CompanyOuterAdapter
```

---

### 4. 配置 Maven 依赖

在 `adapter-company/pom.xml` 中添加依赖配置：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>cn.javayong</groupId>
        <artifactId>platform-sms-adapter</artifactId>
        <version>0.7.2</version>
    </parent>

    <artifactId>adapter-company</artifactId>
    <name>adapter-company</name>
    <description>Company 渠道适配器</description>

    <dependencies>
        <!-- 核心适配器依赖 -->
        <dependency>
            <groupId>cn.javayong</groupId>
            <artifactId>adapter-core</artifactId>
            <version>${project.parent.version}</version>
            <scope>provided</scope>
        </dependency>

        <!-- 添加自定义依赖 -->
        <!-- 例如：Company SDK -->
        <!--
        <dependency>
            <groupId>com.company</groupId>
            <artifactId>company-sms-sdk</artifactId>
            <version>1.0.0</version>
        </dependency>
        -->

        <!-- 其他必要依赖 -->
    </dependencies>

    <build>
        <plugins>
            <!-- 编译插件配置 -->
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <source>1.8</source>
                    <target>1.8</target>
```

## 注意事项

1. **SPI 命名规范**：
   - SPI 配置文件中的 key 值（如 `company`）必须与 `@SPI` 注解的值一致
   - 示例：`@SPI("company")` 对应配置文件中的 `company=...`

2. **枚举值一致性**：前端、后端枚举、SPI 配置中的渠道标识符需要保持一致

3. **依赖范围**：`adapter-core` 依赖使用 `provided` 作用域，避免重复打包

4. **版本管理**：确保新模块的 `parent.version` 与父项目版本一致

5. **测试验证**：完成适配器开发后，需要进行功能测试和集成测试

## 本地启动流程

### 开发模式启动

#### 1. 启动前端服务

```bash
# 进入前端项目目录
cd platform-sms-ui

# 安装依赖（首次执行或依赖有更新时）
npm install

# 启动开发服务器
npm run dev
```

前端服务启动后，默认访问地址为：`http://localhost:9528`

#### 2. 启动后端服务

在 IDEA 中找到启动类 `SmsAdminApplication`，直接运行即可：

```
platform-sms-admin/src/main/java/cn/javayong/platform/sms/admin/SmsAdminApplication.java
```

---

### 生产打包部署

#### 1. 前端打包

```bash
# 进入前端项目目录
cd platform-sms-ui

# 执行打包命令
npm run build
```

打包完成后，静态资源会输出到 `platform-sms-admin/src/main/resources/public/` 目录。

#### 2. 后端打包

在 IDEA 中执行 Maven 打包：

1. 打开 IDEA 右侧 Maven 工具窗口
2. 展开 `platform-sms` 项目
3. 展开 `Lifecycle` 生命周期
4. 双击执行 `clean` 和 `package`

或者在项目根目录执行 Maven 命令：

```bash
mvn clean package -DskipTests
```

打包完成后，可执行 JAR 包位置：

```
platform-sms-admin/target/platform-sms-admin-0.7.2.jar
```

#### 3. 运行打包后的服务

```bash
# 进入 target 目录
cd platform-sms-admin/target

# 运行 JAR 包
java -jar platform-sms-admin-0.7.2.jar
```
